## Overview

Muse 是面向自媒体创作者的本地优先创作工厂。视觉基调为「安静的工作室」：
米白纸感底色 + 白色卡片承载内容，紫罗兰（violet）作为唯一品牌强调色，
表达「灵感 / 缪斯」的联想；界面全中文，信息密度中等，长文阅读行高 1.8。

## Colors

- **Primary `#7C3AED`（violet-600）**：主按钮、激活导航、链接强调、选中态。hover 加深为 `#6D28D9`。浅紫 `#F3EFFF` 用于选中背景与次级按钮。
- **中性色**：背景 `#FAF9F7`（暖米白），卡片表面纯白，正文 `#1C1917`，次要文字 `#78716C`，边框 `#E7E5E4`，浅灰底 `#F5F4F2`。
- **语义色**：success 绿（已清洗/已发布）、warning 琥珀（待处理/审阅中）、danger 红（失败/删除），各配 soft 底色用于 Badge。
- **对比度**：正文与背景 ≥ 12:1；muted 文字仅用于辅助信息（≥ 4.5:1）；白字按钮仅配 primary/danger 深底。

## Typography

系统中文字体栈（PingFang SC 优先），不引入网络字体，保证本地优先与加载速度。
页面标题 20px bold；卡片标题 14px semibold；正文 14px、行高 1.8（长文编辑器同）；
辅助说明 12px；时间戳等微文本 10px。代码/平台正文预览用等宽字体 12px。

## Layout

- ≥768px：左侧固定 208px 侧边导航（模块入口 + 闭环流程提示），右侧内容区最大宽度 64rem 居中。
- <768px：侧栏收起为置顶 56px 头部栏（品牌 + 汉堡按钮），点击打开左侧抽屉导航（宽 16rem、最大 80vw，带遮罩，路由变化或 Esc 自动收起）。
- 列表页卡片栅格：<640px 单列，≥640px 2 列（间距 0.75rem）；统计块 <640px 2 列、≥768px 4 列。
- 写作工作台：<1024px 主画布与工作流面板纵向堆叠；≥1024px「主画布 + 20rem 面板」双栏；≥1280px 面板放宽至 22rem。
- 页头「标题 + 操作」行在窄屏改为纵向堆叠；宽表格（如复盘数据）在卡片内横向滚动，页面本身不出现横向滚动。
- 表单元素间距 0.5rem，行内表单用 flex + gap，窄屏允许换行。

## Elevation & Depth

层级靠边框与浅阴影表达：卡片 `border + shadow-sm`；hover 将边框变为 primary 色；
不使用大面积阴影与叠层模糊。焦点态用 2px primary outline（offset 2px）。

## Motion

- 页面切换在导航开始时先用 120ms、4px 的轻微上移淡出和顶部细进度条回应，新页面再以 220ms 上移淡入接棒；不做大幅滑动或全屏遮罩。
- AI 操作按钮在 pending 时原位显示闪光图标与流光，结果提示用 180ms 缩放淡入；AI 产物区域仅在结果签名真正变化时重播 reveal，首次进入页面不重复动画；编辑器和无关区域保持可交互。
- 工作台 tab 内容使用 180ms 淡入位移，按钮按下缩放至 0.97；动画仅使用 opacity 与 transform。
- 窄屏抽屉导航 220ms 自左滑入，遮罩 120ms 淡入；关闭为即时移除，不做退出动画。
- 所有动效遵守 `prefers-reduced-motion`，降级为无位移、无循环动画的即时状态变化。

## Shapes

卡片圆角 0.75rem，按钮/输入框 0.5rem，Badge 全圆角胶囊。整体软几何，无锐角装饰。

## Components

- **Button**：default（紫底白字）/ secondary（浅紫底紫字）/ outline / ghost / danger 五种；尺寸 sm/default/lg。
- **Card**：Header（标题+描述）+ Content；列表项与表单容器统一用卡片。
- **Badge**：状态语义色胶囊（清洗状态、文章状态、发布状态、审阅严重度、标签）。
- **Input / Textarea / Select / Label**：白底 + 边框 + primary 焦点 outline；Label 为 12px muted。
- **Tiptap 编辑器（沉浸式写作）**：白卡片内嵌；顶部单行紧凑工具栏（撤销/重做、H1-H3、加粗/斜体/删除线/行内代码、三类列表、引用、代码块、插图、预览/导入/导出、专注模式），图标按钮 28px 见方，激活态浅紫底；底部状态栏显示字符数/词数、保存状态与快捷键提示。
- **保存状态**：dirty「有未保存更改…」/ saving「自动保存中…」/ saved「已自动保存」（success 绿）/ error「自动保存失败，稍后重试」（danger 红 + 重试链接）。自动保存只写工作稿，显式「保存为新版本」才产生版本检查点；刷新后若工作稿领先版本，顶部显示琥珀色恢复提示条。
- **Bubble Menu（选中文字浮出）**：tippy 定位在选区上方，白底、细边框、shadow-md；含 B/I/S/行内代码/链接（⌘K）/H2/H3 与紫色 AI 扩写/改写/重组入口；选中公式节点时切换为 LaTeX 输入框 + 更新/删除；在代码块内不显示。
- **`/` 插入菜单**：输入 / 弹出 16rem 列表（标题、列表、任务、引用、代码块、行内/块级公式、表格、图片、分隔线），↑↓ 选择、Enter 确认、Esc 关闭，支持中文与拼音关键词过滤；紧贴光标、超出视口时翻转到上方。
- **AI 改写流**：pending 与结果预览使用编辑器上方位置固定的浅紫卡片（不随选区浮动）；预览含原文摘录、可滚动结果、接受并替换 / 取消；请求期间可继续编辑，原选区经 ProseMirror mapping 跟随，被删除时禁用接受并警示，失败绝不改动正文。
- **代码块**：深色（#1C1917 底 #E7E5E4 字）等宽 12.8px、行高 1.7、横向滚动；右上角语言下拉（27 种语言，hover/聚焦时不透明），lowlight 高亮采用低饱和紫/绿/黄/蓝配色。
- **公式**：KaTeX 渲染；行内公式随文本流，块级公式居中独立块并可横向滚动；hover 浅紫底，选中 2px 紫 outline；空公式显示虚线占位提示。
- **表格**：等分列宽、表头浅灰底、单元格 1px 边框；选中单元格叠浅紫遮罩；窄屏整表随正文缩放，单元格内文字断行。
- **专注模式**：隐藏文章页头、标签行与右侧面板，画布居中收窄至 48rem、最小高度 70vh，只保留工具栏与状态栏；再次点击图标退出。
- **移动端编辑（<768px）**：工具栏自动换行、按钮保持 28px 触控目标；Bubble Menu 与 / 菜单同桌面逻辑；表格与代码块在自身容器内横滚，页面不产生横向滚动。
- **AI 溯源表面**：AI 生成内容均标注来源（版本备注「AI 初稿」、审阅记录 🤖 前缀、mock 模式全局 Badge 提示）。
- **首页与创建向导**：首页首屏只保留两张主卡——「继续上次创作」（标题 + readiness 状态 Badge + 下一步文案）与紫边「开始一次新创作」；下方最多 3 个进行中创作行（标题 + 状态 + 下一步）。首次引导为紫边卡片：3 步单选（每题 2-3 个选项卡片式按钮），「这题跳过 / 全部跳过」常驻，只读示例流程折叠展示且不写数据。创建向导 /create 用三个等宽 tab 起点（想法/资料/经验），AI 方向候选为卡片（推荐 Badge + 与已有创作相似的 warning 提示条），创作说明问题全部预填默认值，主按钮「开始创作 →」次按钮「全部用默认值开始」。
- **创作旅程步骤条**：文章页头下方一行「方向 → 写作 → 检查 → 发布准备 → 已发布 → 复盘」；当前步紫底白字（aria-current="step"），已过步骤前置 ✓，未来步骤 muted；每步可点击直达（面板 / 平台稿页 / 发布记录 / 复盘）。取代旧的写作/审阅/包装/平台版本四 tab；独立审阅页与包装页已重定向到写作台对应面板。
- **创作旅程导航反馈**：方向/写作/检查用 URL `panel` 表达可恢复目标，并与当前面板双向同步；激活后将资料面板、编辑器或审阅面板滚动到可见位置，写作同时聚焦编辑器。重复激活当前步骤也应再次反馈；滚动遵循 `prefers-reduced-motion`。回看只改变视图，不改变 readiness 当前步、正文、版本或保存状态。
- **全局命令面板（⌘K）**：全站 ⌘K / Ctrl+K、桌面侧栏「搜索… ⌘K」按钮与移动端顶栏搜索图标唤起；顶部居中白卡片（宽 min(40rem, 100vw-2rem)，top 10vh），上为搜索输入行（combobox + aria-activedescendant），中为分组结果列表（listbox，max-h 55vh 内滚动），下为快捷键提示行。分组固定顺序「动作 → 文章 → 素材 → 选题 → 复盘经验」；条目三行式——标题（14px 截断）、自然语言状态（11px 紫色，文章为 readiness 状态 + 下一步）、命中片段（12px muted 截断）；激活项浅紫底 + 尾部回车图标。空查询显示动作（开始一次新创作 / 继续上次创作 / 打开设置）与「最近更新」；无结果显示引导文案 + 「去新建一次创作」；查询失败在面板内显示中文报错 + 「重试」项。焦点圈闭于对话框（背景 aria-hidden），Esc 关闭并把焦点还原到唤起前元素（编辑正文时回到编辑器，未保存输入不受影响）；↑↓ 循环选择、Enter 直达页面或写作台 `?panel=` 面板。快捷键优先级：编辑器内选区非空时 ⌘K 仍为链接编辑（编辑器侧拦截），其余场景一律唤起面板。面板全部查询走只读 GET，不产生任何写库。
- **编辑器渐进披露**：工具栏默认仅常用工具（撤销重做、H2/H3、加粗斜体、两类列表、引用、插图、图文预览、专注）；H1/删除线/行内代码/任务列表/代码块/Markdown 源码/导入导出收进「更多 ▾」菜单（role=menu，点外关闭）。自动保存状态只显示绿色「已保存」或红色错误+重试，中间态不显示文案。AI 覆盖性修改被接受后工具栏右侧出现「撤销本次 AI 修改」。
- **术语规范**：普通界面禁止出现「检查点 / 工作稿 / mock / sourceVersionId」；使用「已保存版本 vN / 有未保存的新修改 / 当前正文 / 本地演示 / 本地兜底」等自然语言，技术细节仅进设置或调试视图。
- **发布助手与发布记录**：平台稿卡片底部内嵌可折叠「发布助手」，三步纵向排列——①一键复制整稿/标题/标签/CTA（复制成功按钮短暂变「已复制 ✓」，clipboard 不可用时回退手动复制弹窗）；②下载正文与本地图片（无图片时用灰字说明，不放不可用按钮）；③链接输入 + 紫色主按钮「标记已发布」。发布记录页为只读卡片列表：已发布行主行动「记录这次表现 →」，遗留任务加「历史任务」Badge 仅可删除。
- **复盘向导（记录这次表现）**：顶部上下文条自动带入平台 Badge + 文章标题 + 平台稿名 + 发布链接，不出现任何内部 ID；五步分段（表现数据 → 读者关注 → 假设验证 → 下一次 → 确认摘要），每步一张卡片一个主按钮；摘要为可编辑 textarea，固定「表现观察 / 暂时支持的假设」措辞。复盘经验列表每条含自然语言溯源行（平台 · 文章 · 基于已保存版本 vN · 创作说明 · 发布链接 · 已转为新方向）与「在新创作中复用 →」次级按钮。
- **NextAction 条（成稿就绪）**：写作画布顶部第一行为自然语言状态 Badge（就绪 success / 未就绪 warning）+ 唯一主行动按钮 + 阻塞原因或「可跳过（风险：…）」说明 + 右侧「全部待办（N）」展开；展开列表每条待办用「阻碍发布」（danger）/「建议」（default）分级，附原因、跳过风险与直达按钮；创作说明未确认对齐时提供「确认正文已对齐」。状态只来自事实计算（lib/readiness），不显示 articles.status；发布被服务端拦截时平台版本页顶部显示 danger 横幅「发布已被拦截：原因」。
- **活动修订状态**：写作画布顶部用一行可换行的紧凑 Badge 显示「当前工作稿 / 检查点 vN / 审阅 / 包装 / 平台稿」。正文一有变化，检查点立即回到“尚未形成”，旧下游产物使用 warning 色“已过期”；建立或复用检查点后再恢复 success 色“最新/当前结果”。旧产物不隐藏、不覆盖。
- **证据引用（这句话有什么依据）**：正文中已关联依据的文字使用 1.5px 紫色点线下划线（偏移 3px），hover 浅紫底，点击后右侧资料面板高亮对应依据卡片。资料面板分「查找相关资料 / 本文依据 / 相关素材」三段；依据卡片用语义 Badge 表达有效状态——「依据有效」（success）、「来源已变化」（warning，展开显示引用时快照供核对）、「来源已删除」（danger，保留摘录快照并提示补充依据）。事实检查结论 Badge：「资料支持」success、「缺少资料」default（中性，绝不表述为错误）、「资料冲突」danger、「来源不可用」warning。
- **可编辑 Brief**：选题卡与写作台素材面板共用可收起的 Brief 卡片；字段按「读者/目标 → 主张 → 角度/语气 → 平台 → 要点/大纲 → 要点证据」分组。已有正文时常驻 warning 提示“可能需重新对齐”，但不自动覆盖。AI Brief 与新初稿使用浅紫预览卡，必须“应用到表单/确认保存新版本”才写入，放弃不改变数据。

## Do's and Don'ts

- Do 保持紫色为唯一品牌色，语义色只用于状态。
- Do 所有界面文案使用中文；平台名等专有名词保留原文（X）。
- Do 修改令牌时同步更新 `src/app/globals.css` 的 `@theme` 变量。
- Don't 引入新的强调色、网络字体或大面积阴影。
- Don't 在卡片外直接平铺表单控件。
